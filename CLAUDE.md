# CLAUDE.md — Инструкции для AI

## Общая цель проекта
**Wildberries Product Analysis** — инструмент для селлеров Wildberries.  
Собирает данные из нескольких источников (WB кабинет, MPStats, Google Sheets — лист **Unit**), объединяет их и проводит глубокий **ИИ-анализ** товаров и рекламных кампаний.
Пиши все вопросы и результаты на русском языке - я понимаю только этот язык

## Ключевые возможности
- Глубокий анализ товара по артикулу
- Сбор и анализ статистики (продажи, выручка, позиция, реклама и т.д.)
- Планируется: мощный анализ рекламных кампаний
- Экспорт / работа с Google Sheets
- ИИ-анализ (Groq) на основе собранных данных

## Целевая аудитория
Селлеры Wildberries (продавцы на маркетплейсе)

## Текущий статус
- Хорошо работает вытягивание данных из источников
- Основной фокус сейчас — улучшение анализа и добавление новых типов анализа

## Технологии
- Next.js 15 (App Router) + TypeScript
- Tailwind + shadcn/ui
- Groq (основной LLM для анализа)
- MPStats API
- Wildberries API
- Google Sheets API

## Критически важные правила
- **ЗАПРЕЩЕНО** вносить любые изменения в Google Sheets таблицы (особенно лист Unit). Только чтение.
- Никогда не коммить `.env.local`
- Все API-ключи и токены хранятся **только** в `.env.local`
- Не удалять и не переименовывать важные файлы в `/src/lib/`

## ПРАВИЛО РАБОТЫ С WILDBERRIES API (ВЫСШИЙ ПРИОРИТЕТ)

При любой работе с Wildberries API (content-api, statistics-api, marketplace-api, seller-analytics-api и любыми другими):

### Шаг 1 — всегда читай WB_API.md первым
**Перед написанием любого запроса к WB API** открой файл `WB_API.md` в корне проекта.
- Если нужный эндпоинт там есть со статусом ✅ — используй его точно, без изменений.
- Если эндпоинт есть со статусом ⏳ — уточни у пользователя недостающую информацию.
- Если эндпоинта нет — смотри шаг 2.

### Шаг 2 — изучи существующий код проекта
Посмотри похожие запросы в `/src/lib/` и `/src/app/api/`. Если нашёл — используй как основу.

### Шаг 3 — при сомнении спрашивай, не угадывай
Если ты **не на 100% уверен** в эндпоинте, параметрах или структуре ответа — **НИКОГДА НЕ УГАДЫВАЙ**. Задай вопрос:
- «Какой актуальный body для `/api/v2/nm-report/detail`?»
- «Какая структура ответа у `/api/analytics/v3/sales-funnel/products`?»

### Шаг 4 — всегда веди записи в WB_API.md
После каждого подтверждённого (проверенного тестом) эндпоинта **обязательно обнови `WB_API.md`**:
- Добавь новую запись или обнови существующую ⏳ → ✅
- Запиши точный body запроса и структуру ответа
- Укажи дату подтверждения

Это правило имеет **высший приоритет** в CLAUDE.md.  
Лучше спросить, чем сделать ошибку или использовать устаревшую информацию.
